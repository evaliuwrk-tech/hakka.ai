# 哈客網路學院 數據監測平台 — 更新說明

## 這是什麼

供內部例會使用的監測儀表板,分三個視圖:

| 視圖 | 內容 |
|------|------|
| **月報** | 選擇月份 → GA4 指標卡(與前月比較)、流量管道、裝置、熱門頁面 Top10、FB/IG 分列成效、貼文主題分類統計(圖+可切換平臺的明細表) |
| **週比較** | 選擇週次(週一~週日)→ 自動與上一週比較:GA4(日均活躍、工作階段、瀏覽量)+ FB/IG 分列六指標、對照表、該週熱門頁面 Top10、近 13 週趨勢 |
| **日報** | 選擇日期 → 當日逐篇貼文卡片(依互動排序、標記本日最佳、觸及相對近30日平均倍數)+ 近 14 日每日最佳貼文一覽表 |
| **年度趨勢與KPI** | GA4 近 12 個月趨勢、FB/IG 分列月趨勢與互動組成、社群 KPI 達成情形 |

資料來源:GA4(資源 322296549)+ `FB_IG_流量報告_統計圖表.xlsx`。

**線上網址(例會直接開這個)**:https://evaliuwrk-tech.github.io/hakka.ai/
(GitHub 倉庫:https://github.com/evaliuwrk-tech/hakka.ai,推送 main 分支即自動更新,約 1~2 分鐘生效)

## 檔案結構

```
監測平台/
├── index.html                儀表板網頁(版面與圖表邏輯,平常不用動)
├── data.js                   資料檔(每月/每週更新時重新產生)
├── tools/
│   ├── aggregate_social.py   FB/IG Excel 彙總(逐月×主題×週次)
│   └── build_data.py         產生 data.js(GA4 常數內嵌於此檔)
└── 更新說明.md               本文件
```

## 更新流程

在 Claude Code 對這個資料夾說一句話即可,例如:

> 更新監測平台數據(週會前:更新到最新週;月會前:更新到 115 年 7 月)

Claude 會自動:

1. 透過 analytics-mcp 抓取 GA4 最新數據(月總覽、管道、裝置、熱門頁面、每日數據),
   更新 `tools/build_data.py` 內的 GA4 常數
2. 執行 `tools/aggregate_social.py` 重新彙總 FB/IG Excel
   (前提:`FB_IG_流量報告_統計圖表.xlsx` 已更新;FB/IG 兩張工作表照原格式填)
3. 執行 `tools/build_data.py` 重新產生 `data.js`
4. 推送到 GitHub,GitHub Pages 自動更新上線

手動跑也可以:

```
python 監測平台/tools/aggregate_social.py social.json
python 監測平台/tools/build_data.py social.json
```

## 口徑備忘

- 使用者人數一律採「活躍使用者(activeUsers)」;週比較之活躍使用者為「日均」
  (每日活躍平均;跨日去重的精確口徑以月數據為準)
- 流量管道:firstUserDefaultChannelGroup × activeUsers
- 熱門頁面:細分頁面標題自 114年12月 起才有,先前月份無明細
- 社群互動:FB/IG 平臺原生值,非按讚+留言+分享之和(主題統計同此口徑)
- 週次:以貼文「發佈日期」歸屬,週一~週日,僅列完整週
- KPI 達成率 = 當月 FB+IG 合計 ÷ 年度目標值(基線 +5%),與月報 KPI 追蹤表一致
- GA4 數值會回溯微調,與歷史報告快照有 0.1~0.3% 落差屬正常
